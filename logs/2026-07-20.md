## 🧠 Challenge of the Day

### Deep Clone Object

**Difficulty:** Medium
**Topic:** Objects / Recursion

**Problem:**

Deep-clone an object (including nested objects and arrays) without using `JSON.parse(JSON.stringify())` or `structuredClone()`.

**Example:**

```
Input: { a: 1, b: { c: 2, d: [3, 4] } }
Output: a new object with the same structure but no shared references
```

**Hint:**

Recursively clone. Handle arrays, plain objects, and primitives. Watch out for null (typeof null === 'object').

<details>
<summary>🔍 Reveal solution</summary>

```javascript
function deepClone(obj) {
  if (obj === null || typeof obj !== 'object') return obj;
  if (Array.isArray(obj)) return obj.map(deepClone);
  const clone = {};
  for (const key in obj) {
    if (Object.hasOwn(obj, key)) {
      clone[key] = deepClone(obj[key]);
    }
  }
  return clone;
}
```

**Explanation:** Handles the common cases. Real-world deep clones also need to handle Dates, Maps, Sets, RegExps, circular refs — that's why `structuredClone()` exists in modern JS.

</details>

## 💡 Tip of the Day

**`Object.entries()` for key-value iteration**

Iterating an object's entries is cleaner than `for...in` + `hasOwnProperty` check:

```js
const obj = { a: 1, b: 2, c: 3 };

// ✅ Clean
for (const [key, value] of Object.entries(obj)) {
  console.log(key, value);
}

// Transform to array
const doubled = Object.fromEntries(
  Object.entries(obj).map(([k, v]) => [k, v * 2])
);
```
