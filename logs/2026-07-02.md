## 🧠 Challenge of the Day

### Binary Search

**Difficulty:** Medium
**Topic:** Search Algorithms

**Problem:**

Given a sorted array and a target value, return the index of the target (or -1 if not found). Must be O(log n) time.

**Example:**

```
Input: nums = [1, 3, 5, 7, 9, 11], target = 7
Output: 3
```

**Hint:**

Maintain left and right pointers. Compute mid, compare, halve the search space each iteration.

<details>
<summary>🔍 Reveal solution</summary>

```javascript
function binarySearch(nums, target) {
  let left = 0, right = nums.length - 1;
  while (left <= right) {
    const mid = (left + right) >> 1;  // bit-shift for integer division
    if (nums[mid] === target) return mid;
    if (nums[mid] < target) left = mid + 1;
    else right = mid - 1;
  }
  return -1;
}
```

**Explanation:** Halving the search space each iteration gives O(log n). The bit-shift `>> 1` is a fast integer divide by 2.

</details>

## 💡 Tip of the Day

**Use `Array.prototype.findLast()` and `.findLastIndex()`**

ES2023 added `findLast` / `findLastIndex` — find the LAST matching element without reversing:

```js
const arr = [1, 2, 3, 4, 5, 4, 3];

// Old way
[...arr].reverse().find(x => x === 4);  // mutates copy, awkward

// New way (ES2023)
arr.findLast(x => x === 4);  // 4 (the second occurrence)
arr.findLastIndex(x => x === 4);  // 5
```
