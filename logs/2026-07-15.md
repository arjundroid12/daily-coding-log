## 🧠 Challenge of the Day

### Throttle Function

**Difficulty:** Medium
**Topic:** Closures / Async

**Problem:**

Implement a `throttle(func, limit)` function that ensures `func` is called at most once per `limit` ms.

**Example:**

```
const log = throttle(console.log, 1000);
// Calling log() 10 times in 1 second → log runs only once (or twice, edge case)
```

**Hint:**

Track the last invocation time. If now - last >= limit, call it; otherwise skip (or schedule a trailing call).

<details>
<summary>🔍 Reveal solution</summary>

```javascript
function throttle(func, limit) {
  let lastCall = 0;
  return function (...args) {
    const now = Date.now();
    if (now - lastCall >= limit) {
      lastCall = now;
      func.apply(this, args);
    }
  };
}
```

**Explanation:** Unlike debounce (which waits for silence), throttle guarantees a max rate. Perfect for scroll/mousemove handlers.

</details>

## 💡 Tip of the Day

**Tagged template literals for safe HTML**

Template literals can be tagged with a function that pre-processes parts:

```js
function html(strings, ...values) {
  return strings.reduce((acc, str, i) => {
    const val = values[i] ?? '';
    return acc + str + String(val).replace(/[<>&]/g, c =>
      ({ '<': '&lt;', '>': '&gt;', '&': '&amp;' }[c]));
  }, '');
}

const userInput = '<script>alert(1)</script>';
const safe = html`<div>${userInput}</div>`;
// <div>&lt;script&gt;alert(1)&lt;/script&gt;</div>
```

This is how lit-html and other libraries escape user content safely.
